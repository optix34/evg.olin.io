iButton GeoFence – привязка географической точки к iButton датчику

Версия: 1.0
Для PILOT GPS/GLONASS
Лицензия: Apache 2.0

Описание

Расширение позволяет:

  • Загружать реальный список транспортных средств из PILOT (/ax/tree.php?vehs=1&state=1).
  • Для каждого ТС отображать его iButton ID (если поле присутствует в данных).
  • Создавать, редактировать и удалять привязки «iButton ID → географическая точка (широта, долгота)».
  • При выборе ТС из левого дерева – автоматически центрировать онлайн-карту на привязанной точке и показывать маркер.
  • Все привязки хранятся в localStorage браузера (не требуют бэкенда).

Установка и настройка

  1. Разместите папку расширения на любом статическом хостинге (Cloudflare Pages, GitHub Pages, VPS).
    Убедитесь, что файл Module.js доступен по прямому URL.
  2. В интерфейсе PILOT (администрирование) зарегистрируйте расширение:
    • Имя (slug): ibutton_geofence (только латиница, нижний регистр, без дефисов)
    • Базовый URL: https://ваш-хост/ibutton_geofence/
  3. После регистрации перезагрузите PILOT – слева появится вкладка iButton GeoFence с иконкой ключа.
  4. При первом запуске создайте привязки через интерфейс (кнопка «Добавить»).

Использование

  1. В левой панели выберите транспортное средство, у которого в данных PILOT есть iButton ID.
  2. Если для этого ID существует привязка – карта автоматически переместится к заданной точке и появится маркер с описанием.
  3. Если привязки нет – вы увидите сообщение «Для iButton ID ... точка не задана».
  4. Управляйте привязками в правой панели: добавляйте, редактируйте, удаляйте.
  5. В информационной панели отображается статус выбранного ТС, а кнопка «Показать на карте» повторно центрирует карту.

Примечания для разработчиков

Соответствие AI_SPECS.md:
  • Ext.define('Store.ibutton_geofence.Module', { extend: 'Ext.Component', ... })
  • Метод initModule – точка входа, нет глобальных функций
  • Вкладка добавлена через skeleton.navigation.add(navTab), имеет iconCls: 'fa fa-key'
  • Данные ТС загружаются из реального API PILOT (/ax/tree.php) – нет хардкода
  • doc/index.html – статический, без <script>
  • Взаимодействие с картой через window.mapContainer с проверкой методов

Возможные проблемы

  • Не загружается список ТС: проверьте доступность /ax/tree.php?vehs=1&state=1. Если PILOT на другом домене – потребуется настройка CORS или прокси.
  • Карта не центрируется: убедитесь, что расширение запущено в разделе «Онлайн» (онлайн-карта доступна). Иногда window.mapContainer инициализируется позже – попробуйте переключиться на другую вкладку и обратно.
  • Маркер не появляется: некоторые сборки PILOT не имеют метода addMarker. Расширение пытается использовать прямой Leaflet API, но может не сработать. В этом случае маркер не добавляется, но центрирование карты работает.
  • Поле iButton ID не отображается: возможно, в вашей версии PILOT поле называется иначе (например, driver_key). При необходимости измените в коде record.get('ibutton') на нужное имя.
