Sensor Trigger Map Move Extension

Version: 1.0.0

Author: PILOT Extensions AI

Purpose

This extension allows PILOT users to define trigger points on the map associated with specific sensor IDs. When a vehicle’s sensor (simulated for demo) triggers, the vehicle icon moves to the predefined point – useful for geofencing alerts, maintenance actions, or automated responses.

Features

  • Displays real vehicles loaded from PILOT (/ax/tree.php?vehs=1&state=1).
  • Left navigation tab with vehicle grid (name, VIN, model, year).
  • Right floating panel with:
    • Toolbar: "Add Trigger Point" and "Simulate Trigger".
    • List of saved trigger points (edit/delete).
    • Event log showing trigger actions.
  • Click on map to set coordinates when adding a trigger point.
  • Trigger points stored in localStorage (persistent across reloads).
  • Simulate trigger: pick a vehicle and a sensor ID → vehicle marker moves to the matching trigger point.
  • Demo sensors: each vehicle has mock sensors (engine_temp, fuel_level, door_status) for illustration.

How to Replace Simulation with Real Sensor Events

The extension is designed for easy integration with real sensor APIs. In Module.js, locate the function simulateTrigger(vehicleId, sensorId). Replace the call to this function with a real-time event listener (e.g., WebSocket, polling) that calls the same method:

// Inside your real-time event handler:
var extension = Ext.ClassManager.get('Store.sensortrigger.Module');
if (extension && extension.moveVehicleOnSensor) {
    extension.moveVehicleOnSensor(vehicleId, sensorId);
}

The method moveVehicleOnSensor(vehicleId, sensorId) checks for a stored trigger point with sensorId and moves the vehicle marker.

Installation & Verification

  1. Upload the sensortrigger/ folder to a public HTTPS host (Cloudflare Pages, GitHub Pages, etc.).
  2. Ensure https://your-host.com/sensortrigger/Module.js is accessible directly in a browser.
  3. In PILOT admin, register the extension with name sensortrigger and base URL https://your-host.com/sensortrigger/.
  4. Reload PILOT – a new tab "Позиция по метки" appears in the left navigation and a control panel on the right side of the map.

Limitations / Notes

  • The extension uses the existing Online map (window.mapContainer). It does NOT create a new map.
  • Trigger points are stored per browser (localStorage).
  • Vehicle markers are moved by updating the Leaflet marker position – works with PILOT’s native markers.
  • If a marker does not exist for a vehicle, a new one is created at the target point.
  • All API calls to PILOT endpoints respect CORS through the built-in proxy (/store/sensortrigger/...).
