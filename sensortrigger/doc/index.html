📌 Marks Manager Extension for PILOT

Version 1.0 localStorage

Overview

The Marks Manager extension allows PILOT users to create, store, and manage custom marks (bookmarks) associated with vehicles from the PILOT system. Marks are saved locally in your browser's localStorage and persist across sessions.

Features

  • ✅ Load real vehicles from PILOT (/ax/tree.php)
  • ✅ Add new marks with unique ID, vehicle association, description, and coordinates
  • ✅ Edit and delete existing marks
  • ✅ View mark details in the main panel
  • ✅ Optional “Show on Map” button centers the PILOT map on the mark location
  • ✅ Stores last fix time (creation/update timestamp)
  • ✅ Fully client‑side – no backend required

How to Use

  1. Add a Mark: Click the “Add Mark” button in the left panel, fill in the ID, select a vehicle from the dropdown, enter description and coordinates (lat/lon), then save.
  2. View Details: Click any row in the left grid – its details will appear on the right panel.
  3. Update: Select a mark, then click “Update” to edit its fields. The Mark ID cannot be changed after creation.
  4. Delete: Select a mark, then click “Delete” and confirm removal.
  5. Show on Map: If you are in the Online or History section, click “Show on Map” to center the map on the mark's coordinates (requires active map).

Technical Details

Storage key: marks_manager_marks in localStorage

Vehicle data source: /ax/tree.php?vehs=1&state=1 (standard PILOT API)

Integration pattern: Navigation tab + main panel (Pattern A from AI_SPECS.md)

Required PILOT objects: skeleton.navigation, skeleton.mapframe, window.mapContainer / getActiveTabMapContainer() (optional for map centering)

⚠️ Note: This extension does not modify or write any data back to the PILOT server. All marks are stored locally in your browser. If you clear your browser's local storage, all marks will be lost.

Installation & Deployment

To install this extension:

  1. Place the folder marks_manager/ on a public HTTPS host (e.g., Cloudflare Pages, GitHub Pages, or your own web server).
  2. Ensure Module.js and doc/index.html are accessible at the same base URL.
  3. Register the extension in PILOT admin panel: set the extension name to marks_manager and the base URL to the public folder (e.g., https://yourdomain.com/marks_manager/).
  4. PILOT will proxy the extension as /store/marks_manager/Module.js at runtime.

No additional dependencies or build steps are required.
