VIN Insight – расшифровка VIN для PILOT

Версия: 1.0
Автор: PILOT Extensions Generator
Лицензия: Apache 2.0

Описание

Расширение VIN Insight предназначено для получения расшифрованной информации о транспортном средстве по его VIN-номеру с использованием внешнего API auto.dev.

Особенности:

  • Загрузка реального списка автомобилей из PILOT (/ax/tree.php?vehs=1&state=1).
  • Безопасное хранение API-ключа в localStorage браузера.
  • Отображение основных параметров (марка, модель, год, кузов, двигатель и т.д.) и сырого JSON-ответа.
  • Полная отказоустойчивость – отсутствие VIN или ключа не ломает интерфейс.

Установка и настройка

  1. Разместите папку расширения на вашем веб-сервере (например, https://your-server.com/vininsight/).
  2. Убедитесь, что файл Module.js доступен по URL https://your-server.com/vininsight/Module.js.
  3. В интерфейсе PILOT зарегистрируйте расширение, указав:
    • Имя: vininsight
    • Базовый URL: https://your-server.com/vininsight/
  4. Если используется прокси для auto.dev, настройте перенаправление запросов с /store/vininsight/autodev/vin/... на https://auto.dev/api/vin/.... Пример для Nginx: 
    location /store/vininsight/autodev/ {
    rewrite ^/store/vininsight/autodev/vin/(.*)$ /api/vin/$1 break;
    proxy_pass https://auto.dev;
    proxy_set_header Host auto.dev;
}
  5. После регистрации перезагрузите PILOT – слева появится вкладка «VIN Insight» с иконкой автомобиля.

Использование

  1. В верхней панели расширения введите ваш API-ключ от auto.dev и нажмите «Сохранить ключ».
  2. В левом дереве выберите любое транспортное средство из списка PILOT.
  3. Расширение автоматически выполнит запрос к auto.dev и отобразит:
    • Статус расшифровки,
    • Основные характеристики в секции «Обзор»,
    • Полный JSON-ответ в секции «Сырая расшифровка».
  4. Кнопка «Test API» позволяет проверить работу API на тестовом VIN 3GCUDHEL3NG668790 без выбора ТС.

Примечания для разработчиков

Важно: Расширение полностью соответствует AI_SPECS.md:
  • Использует Ext.define('Store.vininsight.Module', { extend: 'Ext.Component', ... })
  • Метод инициализации – initModule (без глобальной функции)
  • Вкладка добавляется через skeleton.navigation.add() и имеет iconCls: 'fa fa-car'
  • Реальные данные загружаются из /ax/tree.php – никаких моков
  • doc/index.html содержит исключительно статическую документацию, без скриптов и логики расширения

Возможные проблемы и их решение

  • Не загружается список ТС: проверьте доступность /ax/tree.php?vehs=1&state=1 из браузера. Если PILOT работает на другом домене – возможно, потребуется настройка CORS.
  • Расшифровка не работает:
    • Убедитесь, что API-ключ сохранён и активен.
    • Проверьте, что прокси-путь autodev/vin/... правильно перенаправляется на auto.dev (см. пример настройки выше).
    • Откройте вкладку «Сеть» в инструментах разработчика, чтобы увидеть точный HTTP-ответ.
  • Иконка вкладки не отображается: PILOT должен подключать Font Awesome 6. Если иконки нет – проверьте, загружается ли шрифт. Можно заменить на стандартную иконку iconCls: 'x-fa fa-car'.
