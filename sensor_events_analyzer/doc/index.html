📡 Sensor Events Analyzer

PILOT Extension – анализ срабатываний датчиков транспортных средств.

📋 Возможности

  • Загрузка реальной иерархии объектов из PILOT (/ax/tree.php).
  • Настраиваемый API‑эндпоинт для получения событий датчиков (сохраняется в localStorage).
  • Отображение списка событий с координатами и возможностью показать на карте.
  • Статистика: общее количество событий, разбивка по типам датчиков, самый активный датчик.
  • График (столбчатая диаграмма) при наличии window.Highcharts.
  • Сырой JSON‑ответ для отладки.
  • Интеграция с существующей картой PILOT (Online / History).

⚙️ Настройка

После установки расширения в PILOT:

  1. Откройте вкладку Sensor Events Analyzer в левой панели.
  2. В верхней панели укажите URL для получения событий, например: /ax/sensor_events.php?vehid= (эндпоинт должен возвращать JSON по запросу ?vehid=123).
    Если в вашей системе такого эндпоинта нет, вы можете реализовать свой бэкенд или использовать прокси.
  3. Нажмите Save URL – настройка сохранится и будет использоваться при следующих загрузках.
  4. Выберите транспортное средство в дереве слева – события загрузятся автоматически.
  5. Для ручного обновления используйте кнопку Reload events.
📌 Примечание: Расширение не создаёт собственных эндпоинтов – оно лишь читает данные из указанного вами URL. Формат ответа должен соответствовать ожидаемому: 
[
  {
    "timestamp": "2025-03-10 14:23:45",
    "sensor_name": "Ignition",
    "sensor_type": "digital",
    "value": "ON",
    "lat": 55.751244,
    "lon": 37.618423,
    "event_id": 12345
  }
]

🗺️ Работа с картой

При клике на строку события или на иконку маркера в таблице:

  • Карта перемещается к координатам события.
  • Добавляется маркер с подсказкой (название датчика и значение).
  • Все маркеры, созданные расширением, автоматически удаляются при выборе другого транспортного средства или перезагрузке событий.

🔧 Требования к окружению PILOT

  • PILOT версии 7.7+ с загруженным Ext JS.
  • Наличие глобальных объектов window.skeleton, window.getActiveTabMapContainer (или window.mapContainer).
  • Опционально: window.Highcharts для графиков.
⚠️ Внимание: Если эндпоинт событий не задан или недоступен, расширение покажет соответствующее сообщение. Убедитесь, что ваш сервер поддерживает CORS или используйте прокси-путь /store/sensor_events_analyzer/backend/ (если вы разместили PHP‑прокси).

📂 Структура установки

sensor_events_analyzer/

После загрузки на хостинг (Cloudflare Pages, GitHub Pages, VPS) зарегистрируйте базовый URL в админке PILOT:
https://ваш-хост/sensor_events_analyzer/
Имя расширения в PILOT должно быть sensor_events_analyzer (snake_case).

🐞 Устранение неполадок

  • Дерево объектов не загружается – проверьте, что /ax/tree.php?vehs=1&state=1 доступен из консоли браузера.
  • События не загружаются – откройте вкладку Network в DevTools, проверьте запрос к вашему эндпоинту, убедитесь в корректности JSON.
  • Карта не реагирует – убедитесь, что вы находитесь в разделе Online или History PILOT, где карта активна.
