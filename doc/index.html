Sensor Dashboard for PILOT GPS

Version: 1.0

Extension name: sensor_dashboard

Description

This extension displays all available sensors and their current values for any vehicle selected from the left navigation tree.

It uses standard PILOT endpoints:

  • /ax/tree.php?vehs=1&state=1 – retrieves the list of vehicles (hierarchical).
  • /ax/state.php?vehid=XXX – retrieves the current sensor values for a specific vehicle.

Features

  • Left panel with vehicle tree (groups + vehicles).
  • Dynamic grid showing all sensor keys and values.
  • Automatic unit formatting using PILOT helpers (speedSS, mileageSS, num) when available.
  • Refresh button to reload data for the selected vehicle.
  • Graceful error handling and loading indicators.

Installation & Hosting

Upload the following files to a public HTTPS host (e.g., Cloudflare Pages, GitHub Pages, or your own server):

sensor_dashboard/
├── index.html
├── Module.js
├── style.css
└── doc/
    └── index.html

After uploading, verify that https://YOUR-HOST/Module.js is accessible. Then in PILOT Admin → Applications → Add, set:

  • Name: Sensor Dashboard
  • URL: https://YOUR-HOST/ (with trailing slash)

Ensure the extension slug is sensor_dashboard (lowercase snake_case). PILOT will proxy it as /store/sensor_dashboard/....

Usage

  1. After loading PILOT, click on the "Sensor Dashboard" tab in the left navigation (icon: microchip).
  2. Expand the vehicle tree and click on any vehicle.
  3. The main panel will display all available sensors and their current values.
  4. Use the "Refresh" button to reload data.

Limitations

  • Sensor list depends on what /ax/state.php returns; not all vehicles may have the same sensors.
  • No historical data – only current values.
  • The tree may show groups/folders without vehicles; they are ignored for sensor loading.
