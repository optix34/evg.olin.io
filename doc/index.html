Расширение PILOT: «Доп. Оборудование»

Версия: 1.0

Имя расширения (slug): sensor_dashboard

1. Назначение

Учёт и визуализация дополнительного оборудования на каждом транспортном средстве: АОГ, Видео, Табло, Голос, ТФ, BLE, ТХГ, ДУТ, Датчик t. Все настройки сохраняются в localStorage браузера.

2. Основные функции

Левая панель – список ТС

  • Отображает все транспортные средства из PILOT (/ax/tree.php).
  • Колонка «Датчики» – иконки включённого оборудования.
  • Поле «Поиск ТС» – фильтр по названию.
  • Выпадающий список «Фильтр по датчику» – показывает только ТС с выбранным датчиком.

Правая панель – настройка датчиков для выбранного ТС

  • Таблица, где каждый датчик – отдельная колонка с чекбоксом.
  • Щелчок по чекбоксу – изменение состояния (включено/выключено).
  • Контекстное меню в заголовке колонки (стрелка):
    • Выбрать все – включить датчик для текущего ТС.
    • Снять все – выключить.
    • Фильтровать по этому датчику – применить фильтр в левой панели.
  • Кнопка «Применить» – сохранить изменения в localStorage и обновить иконки/статистику.

Таблица «Статистика по всем объектам»

  • Показывает для каждого датчика: общее количество ТС, количество включённых и процент.
  • Клик по строке – автоматическая фильтрация левого списка по этому датчику.

Диаграмма (Highcharts)

  • Столбчатая диаграмма: количество ТС с включённым датчиком по каждому типу.
  • Адаптируется под ширину окна, интерактивные подсказки.

4. Сохранение данных

Ключи в localStorage – sensor_dashboard_<vehid>. Данные не синхронизируются между браузерами.

5. Установка и развертывание

Структура расширения:

sensor_dashboard/
├── Module.js
├── index.html          (может быть пустым)
├── style.css           (опционально)
└── doc/
    └── index.html      (этот файл)

Разместите файлы на любом HTTPS-хостинге (Cloudflare Pages, GitHub Pages, VPS). Убедитесь, что Module.js доступен по прямой ссылке: https://ваш-хост/Module.js.

В админ‑панели PILOT → Applications → Add:

  • Name: Доп. Оборудование (любое)
  • URL: базовый URL вашего хостинга (например, https://ваш-хост/)
  • Extension name (slug): sensor_dashboard (обязательно)

6. Требования к окружению

  • PILOT GPS с поддержкой внешних расширений.
  • Загруженные библиотеки: Ext JS 7.7+, Highcharts, Font Awesome.
Примечание: Расширение не использует серверную часть – вся логика на стороне клиента. Никакие данные не отправляются на внешние серверы.

7. Возможные проблемы

  • Не загружается список ТС: проверьте URL в админке PILOT и доступность Module.js.
  • Чекбоксы не сохраняются: убедитесь, что localStorage не заблокирован (очистите кэш).
  • Иконки отсутствуют: на странице PILOT должен быть подключён Font Awesome (обычно есть).
