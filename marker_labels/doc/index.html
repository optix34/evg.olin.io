📍 Marker Labels Manager – PILOT Extension

Version: 1.0
Compatibility: PILOT with Ext JS 7.7+ and extension system.

📌 Description

This extension allows you to create custom markers (labels) and associate them with vehicles from your PILOT fleet. For each marker, the extension displays the last fix time of the linked tracker, helping you monitor vehicle positions.

🚀 Features

  • Left navigation tab with "Add Marker" button.
  • Modal form to create a marker: unique ID, vehicle selection, description, latitude/longitude.
  • All markers stored in browser's localStorage (persist across sessions).
  • Main panel shows a sortable grid with marker details and last fix time.
  • Delete markers individually.
  • "Refresh Last Fix Times" button to reload vehicle data from PILOT.
  • Integration with PILOT's /ax/tree.php to obtain vehicle names and last fix timestamps.

⚙️ Installation

  1. Upload the extension folder to a static web server (Cloudflare Pages, GitHub Pages, etc.).
  2. In PILOT Admin, create a new extension with name marker_labels (exact snake_case).
  3. Set the external base URL to the folder containing Module.js, e.g. https://your-domain.com/marker_labels/.
  4. Save and reload PILOT. The "Marker Labels" tab will appear in the left navigation.
ℹ️ Note: The extension assumes that the endpoint /ax/tree.php?vehs=1&state=1 returns vehicles with a field last_time (Unix timestamp or string). If your PILOT version does not provide this field, the last fix time will show "Not available". You can manually refresh using the button in the main panel.

🧩 Usage

  1. Click Add Marker in the left panel.
  2. Enter a unique Marker ID, select a tracker from the dropdown, add an optional description, and enter latitude/longitude.
  3. Click Save – the marker appears in the main grid.
  4. To delete a marker, click the "Delete" link in the Actions column.
  5. To refresh last fix times, click the "Refresh Last Fix Times" button in the bottom toolbar.

🐛 Troubleshooting

  • Vehicle list is empty: Check that /ax/tree.php is accessible and returns valid JSON with vehid and text.
  • Last fix time not showing: Ensure that the tree response contains last_time or last_fix. If not, the extension will display "Not available".
  • Markers disappear after browser clear: Markers are stored in localStorage; clearing browser data will remove them. Consider exporting markers if needed (not implemented in this version).
  • Navigation tab missing icon: Ensure your PILOT build includes Font Awesome 6. The class fa fa-map-marker-alt should be available.
